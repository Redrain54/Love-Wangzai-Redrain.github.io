token 能放在cookie中吗

  1. 客户端使用用户名跟密码请求登录
  2. 服务端收到请求，去验证用户名与密码
  3. 验证成功后，服务端签发一个 token ，并把它发送给客户端
  4. 客户端接收 token 以后会把它存储起来，比如放在 cookie 里或者 localStorage 里
  5. 客户端每次发送请求时都需要带着服务端签发的 token（把 token 放到 HTTP 的 Header 里）
  6. 服务端收到请求后，需要验证请求里带有的 token ，如验证成功则返回对应的数据

js如何获取/禁用cookie

cookie

  1. name=value 必选参数,分别表示要存入的 属性 和 值。
  2. expires=evalue 可选参数。该对象的有效时间
  3. path=pvalue 可选参数。限制访问 cookie 的目录，默认情况下对于当前网页所在的同一目录下的所有页面有效
  4. domain=dvalue 可选参数。用于限制只有设置了的域名才可以访问；如果没有设置，则默认在当前域名访问
  5. .secure=true|false 可选参数，默认是 true 不安全传输。安全设置，指明必须通过 安全的通信通道来传输（https) 才能获得 cookie,true 不安全，默认值；false 安 全，必须通过 https 来访问
