什么是前端？

  • 解决GUI人机交互问题（大概关注 功能、美观、性能、无障碍、安全、兼容这几个方面）
  • 跨终端
    • PC/移动浏览器
    • 客户端/小程序
    • VR/AR等
  • Web技术栈（HTML-内容、CSS-样式 、JavaScript-行为、HTTP网络协议）
  • 前端工程师就是使用Web技术栈解决多端图形用户界面交互的工程师

HTML相关

1、<!DOCTYPE html>——文档类型。

混沌初分，HTML 尚在襁褓（大约是 1991/92 年）之时，DOCTYPE 用来链接一些 HTML 编写守则，比如自动查错之类。DOCTYPE 在当今作用有限，仅用于保证文档正常读取。现在知道这些就足够了。

More: 标记了html的版本，浏览器会根据这个去决定使用哪种渲染模式。如若不写，浏览器会选择使用比较老旧、怪异的模式去渲染，从而产生不可预料的渲染结果。

2、属性值

推荐使用双引号包裹，但有一些属性值可以省略不写，如required、readonly

3、列表

有ul、ol、dl。分别是无序列表(Unordered List)、有序列表(Ordered List)、定义列表(Definition List)。前两个包括子标签li，即列表项(List Item)。第三个此前并未接触过，其包括了子标签dt（定义列表中的项目）和dd（描述列表中的项目）。

5、引用标签

  • <blockquote>定义块引用（一般用于长引用），标签之间的所有文本都会从常规文本中分离出来，经常会在左、右两边进行缩进（增加外边距），而且有时会使用斜体。也就是说，块引用拥有它们自己的空间。

  • <cite>标签通常表示它所包含的文本对某个参考文献的引用，比如书籍或者杂志的标题。按照惯例，引用的文本将以斜体显示。

    用<cite>标签把指向其他文档的引用分离出来，尤其是分离那些传统媒体中的文档，如书籍、杂志、期刊，等等。如果引用的这些文档有联机版本，还应该把引用包括在一个 <a> 标签中，从而把一个超链接指向该联机版本。

    <cite>标签还有一个隐藏的功能：它可以使你或者其他人从文档中自动摘录参考书目。我们可以很容易地想象一个浏览器，它能够自动整理引用表格，并把它们作为脚注或者独立的文档来显示。<cite>标签的语义已经远远超过了改变它所包含的文本外观的作用；它使浏览器能够以各种实用的方式来向用户表达文档的内容。

  • <q>定义短引用，浏览器一般会给标签内的内容加上双引号。

<q>标签在本质上与 <blockquote>是一样的。不同之处在于它们的显示和应用。<q> 标签用于简短的行内引用。如果需要从周围内容分离出来比较长的部分（通常显示为缩进的块），请使用 <blockquote>标签。

6、<code>标签

常用于插入计算机文本，也即在表示代码文本时可以使用此标签，配合<pre>标签可以实现多行的代码文本。被包围在 pre 元素中的文本通常会保留空格和换行符。而文本也会呈现为等宽字体。

7、内容整体划分的标签（以下全为h5新增标签）

  • <header>：注意与<head>进行区分，这是一个定义文档页眉（介绍信息）的标签，一般包括导航栏、logo图片等内容

  • <nav>即导航标签

  • <main>标签规定文档的主要内容。其内容对于文档来说应当是唯一的。它不应包含在文档中重复出现的内容，比如侧栏、导航栏、版权信息、站点标志或搜索表单。

  • <article>标签规定独立的自包含内容。其潜在来源包括论坛帖子、报纸文章、博客条目、用户评论等。

  • <aside>标签定义其所处内容之外的内容。其内容应该与附近的内容相关。可用作文章的侧栏，比如推荐文章等

  • <footer>标签定义文档或节的页脚。页脚通常包含文档的作者、版权信息、使用条款链接、联系信息等等。可以在一个文档中使用多个

8、语义化是什么？

  • HTML中的元素、属性及属性值都拥有某些含义
  • 开发者应该遵循语义来编写HTML
    • 有序列表用ol，无序列表用ul
    • lang属性表示内容所使用的语言

9、HTML传达内容，而不是样式

10、如何做到语义化？

  • 了解每个标签和属性的含义（推荐阅读MDN文档）
  • 思考什么标签最适合描述这个内容
  • 不使用可视化工具生成代码

11、src和href的区别

一般直接引用嵌入页面当中的用src，比如图片、音视频，而引用外部的链接会用href（hypertext referrence），比如<a>标签、引用css文件的标签等。
