异步（Asynchronous）编程

首先了解一下什么是异步编程，与之相对应的，还存在一个同步的概念。

“同步模式”就是上一段的模式，==后一个任务等待前一个任务结束，然后再执行==，程序的执行顺序与任务的排列顺序是一致的、同步的；”异步模式”则完全不同，每一个任务有一个或多个回调函数（callback），前一个任务结束后，不是执行后一个任务，而是执行回调函数，后一个任务则是不等前一个任务结束就执行，所以程序的执行顺序与任务的排列顺序是不一致的、异步的。

“异步模式”非常重要。在浏览器端，耗时很长的操作都应该异步执行，避免浏览器失去响应，最好的例子就是Ajax操作。在服务器端，”异步模式”甚至是唯一的模式，因为执行环境是单线程的，如果允许同步执行所有http请求，服务器性能会急剧下降，很快就会失去响应。

下面介绍JS中四种异步编程的方法：

  • 回调函数

    假定两个函数f1和f2，后者等待前者的执行结果。

    如果f1是一个很耗时的任务，那么为了提高性能，就可以考虑改写f1，把f2写成f1的回调函数。

    这样，就把同步操作变成了异步操作，f1不会堵塞程序运行，相当于先执行程序的主要逻辑，将耗时的操作推迟执行。

    回调函数的优点是简单、容易理解和部署，缺点是不利于代码的阅读和维护，各个部分之间高度耦合（Coupling），流程会很混乱，而且每个任务只能指定一个回调函数。

  • 其余的(事件监听、订阅/发布、Promise)可参考这个博客

常见的异步编程：

  • fs文件操作

  • 数据库操作

  • AJAX

  • 定时器

Promise是什么？

抽象表达：

1）Promise是一门新的技术（ES6规范）

2）Promise是JS中进行异步编程的新解决方案，旧方案是单纯的回调函数

具体表达

1）从语法上来说，Promise是一个构造函数

2）从功能上来说：promise对象用来封装一个异步操作并可以获取其成功/失败的结果值

为什么要用Promise

指定回调函数的方式更加灵活

1）旧的：必须在启动异步任务前指定

2）promise：启动异步任务 => 返回promise对象 => 给promise对象绑定回调函数（甚至可以在异步任务结束后指定/多个）

支持链式调用，可以解决回调地狱问题

1）什么是回调地狱？

  回调函数嵌套调用，外部回调函数异步执行的结果是嵌套的回调执行的条件

2）回调地狱的缺点？

 不便于阅读、不便于异常处理

3）解决方案？

 promise链式调用

promise的状态

是实例对象的一个属性——PromiseState，具有以下取值：

  • pending 意为未决定的，是初始值
  • resolved / fulfilled 意为成功
  • rejected 意为失败

其状态的改变只有两种，要么pending变为resolved，要么pending变为rejected，且每个对象只能改变一次。无论变为成功还是失败，都会有一个结果数据（PromiseResult，也是实例对象的一个属性）。成功的结果数据一般称为value，失败的结果数据一般称为reason，有且仅有resolve和reject这两个函数可以修改结果数据。

Promise主要的API

  • Promise构造函数: Promise (excutor) {}

    • excutor函数：执行器 (resolve,reject) => {} （==会在Promise内部立即同步调用，异步操作在执行器中执行==）
    • resolve函数：内部定义成功时调用的函数 value => {}
    • reject函数：内部定义失败时调用的函数 reason => {}

  • Promise.prototype.then方法: (onResolved, onRejected) => {}

    • 从上面的代码中不难看到，一般情况下then方法只传入两个回调函数，不再写其函数体。但不可忽略其也可以具备函数体，并且会返回一个新的Promise对象

  • Promise.prototype.catch方法: (onRejected) => {}

    • 只能传入失败的回调函数

  • Promise.resolve方法: (value) => {}。（可以快速的得到、封装一个promise对象）

    • value可以是成功的数据或是promise对象
    • 会返回一个成功/失败的promise对象
    • ==与以下的函数一样，都是静态方法，通过大写的Promise直接调用，不需要创建实例==

  • Promise.reject方法: (reason) => {}

    • reason是失败的原因，也即作为PromiseResult输出
    • 会返回一个失败的promise对象

  • Promise.all方法: (promises) => {}

    • promises是包含n个promise的数组
    • 返回一个新的promise, 只有传参的数组中所有的promise都成功才成功, 只要有一个失败了就直接失败。成功时返回的是参数中每一个promise对象成功时返回结果组成的数组，失败则是参数数组中==第一个失败的==promise对象返回的失败的结果。

  • Promise.race方法: (promises) => {}

    • promises: 包含n个promise的数组
    • 返回一个新的promise, 第一个完成（也即状态从pending改为fullfilled）的promise的结果状态就是最终的结果状态。（也就是race作为赛跑的字面意思）

Promise的几个关键问题

  • 如何改变promise 的状态?

    • resolve(value): 如果当前是pending 就会变为resolved/fulfilled

    • reject(reason): 如果当前是pending 就会变为rejected

    • 抛出异常: 如果当前是pending 就会变为rejected

  • 一个promise 指定多个成功/失败回调函数, 都会调用吗?

    • 当promise 改变为对应状态时==都会调用==

  • 改变promise状态和指定回调函数谁先谁后?

    • 都有可能, 正常情况下是先指定回调再改变状态, 但也可以先改状态再指定回调

    • 如何先改状态再指定回调?

      • 在执行器中直接调用resolve()/reject()
      • 延迟更长时间才调用then()

    • 什么时候才能得到数据?（注意指定回调不意味着执行回调，毕竟执行回调需要先执行resolve函数以拿到数据）

      • 如果先指定的回调, 那当状态发生改变时, 回调函数就会调用, 得到数据（指定回调——状态改变——执行回调）

      • 如果先改变的状态, 那当指定回调时, 回调函数就会调用, 得到数据

  • promise.then()返回的新promise 的结果状态由什么决定?

    • 简单表达: 由then()指定的回调函数执行的结果决定
    • 详细表达:
      • 如果抛出异常, 新promise 变为rejected, reason 为抛出的异常
      • 如果返回的是非promise 的任意值, 新promise 变为resolved, value 为返回的值
      • 如果返回的是另一个新promise, 此promise 的结果就会成为新promise 的结果

  • promise 如何串连多个操作任务?

    • promise 的then()返回一个新的promise, 可以开成then()的链式调用

    • 通过then 的链式调用串连多个同步/异步任务

  • promise 异常穿透?

    • 当使用promise 的then 链式调用时, 可以在==最后指定失败的回调==
    • 前面任何操作出了异常, 都会传到最后失败的回调中处理（可以用.catch()方法）

  • 中断promise 链?

    • 当使用promise 的then 链式调用时, 在中间中断, 不再调用后面的回调函数

    • 办法: 在回调函数中==返回一个pending 状态的promise 对象==

自定义Promise（手写Promise代码）

自己手写Promise，大体可遵循以下步骤进行实现功能：

  • 注意到Promise对象有两个属性，分别是PromiseState和PromiseResult

  • 实现Promise的resolve()和 reject()——要对两个属性进行修改

  • 实现throw抛出异常的功能——在excutor()外围包裹try-catch

  • 实现Promise对象的状态只能修改一次——添加判断条件，在pending状态下直接return

  • 实现then方法的回调——注意是Promise.prototype.then()，传参时onResolved和onRejected，通过状态去判断执行哪一个回调，注意回调传参。

  • ==解决异步任务的回调==——需要新建一个callback对象变量，用以在pending状态下的then()里保存回调函数，从而可以在执行器函数中调用

  • ==实现指定的多个回调（这点是上一点的加强版）==——因为then指定的回调可能有多个，所以单个的callback对象变量就不足以实现多个，于是可以改用callbacks数组变量，然后在pending状态下的then()里进行push操作，再在执行器函数里进行循环遍历执行。

  • 实现同步修改then方法的返回结果——then()方法也会返回一个Promise对象，并且会根据其回调函数的调用而决定其PromiseState属性。所以需要对（成功/失败）回调函数的返回结果进行判断：如果该返回值是一个Promise类型对象，则根据此对象的执行结果修改状态，如果不是Promise对象，则需要将对象的状态相应的改变并将PromiseResult修改为返回值（如果没有返回值则返回undefined）==只需要调resolve(传参)即可==。注意要解决抛出异常问题，所以要加上try-catch语句

  • 实现异步修改then方法的返回结果——这里实现思路综合了==异步任务回调==和上一个同步修改then方法返回结果。需要注意的是this指针指向的问题，同时也别忘了添加try-catch

  • 可以看到，当前我们已经写了很多try-catch并且需要判断返回状态的代码块了，为了简便和后期维护，我们可以将其封装为一个callback函数，并传入一个回调函数作为参数。（注意this指向问题，有对象则其指向对象，否则指向window）

  • 实现catch()方法和异常穿透——可以直接借助前面实现好的then()方法，第一个参数传undefined。然后要在then方法一开始添加默认的回调函数，以实现在调用多个then()（不传第二个失败的回调）时可以只在最后面进行catch()

  • 实现resolve()方法——返回一个Promise对象，判断其参数是都为Promise对象，从而相应的改变其状态。

  • 实现reject()方法——返回一个失败的Promise对象，不管传入的是什么。

  • 实现all()方法——参数是一个Promise对象数组，返回值是第一个失败的Promise对象或全部都成功的对象的结果组成的数组，所以可以用一个变量计算成功状态的对象数目，并有一个数组变量存放成功对象的结果。 ==值得注意的是，不要用数组的push方法，应为输出结果要按照参数的顺序输出，但是执行顺序却不一定。如果用push方法的话，可能会打乱顺序。

  • 实现race()方法——这个比较简单，直接返回一个Promise对象，里面对参数对象数组进行遍历，然后直接修改状态即可，因为Promise对象状态只能修改一次，所以谁先修改状态就返回谁，所以也就啥也不用干。
