0/1背包问题——认识动态规划

发表于2021-10-29|更新于2022-04-30|studyAlgorithm
|字数总计:3.5k|阅读时长:12分钟|阅读量:

何时使用动态规划

  • 重叠子问题：如果一个问题可以被分解为若干个子问题，且这些子问题会重复出现，那么就称这个问题拥有重叠子问题（Overlapping Subproblems）。动态规划通过记录重叠子问题的解，来使下次碰到相同的子问题时直接使用之前记录的结果，以此避免大量重复计算。因此，一个问题必须拥有重叠子问题，才能使用动态规划去解决。
  • 最优子结构：如果一个问题的最优解可以由其子问题的最优解有效地构造出来，那么称这个问题拥有最优子结构(Optimal Substructure)。最优子结构保证了动态规划中原问题的最优解可以由子问题的最优解推导而来。因此，一个问题必须拥有最优子结构，才能使用动态规划去解决。
  • 至此，可以指出，一个问题必须拥有重叠子问题和最优子结构，才能使用动态规划去解决

核心

如何设计状态和状态转移方程，正是动态规划的核心，也是动态规划最难的地方。

如果在题目中分析出状态和选择时，就可以进一步寻找状态转移方程了。

0/1背包

概述

有多个物品，重量不同、价值不同，以及一个容量有限的背包，选择一些物品装到背包中，问怎么装才能使装进背包的物品总价值最大。根据不同的限定条件，常见的背包问题有以下两种：

  • 如果每个物体可以切分，称为一般背包问题，用贪心法球最优解。比如吃自助餐，在饭量一定的情况下，怎么吃才能使迟到肚子里的最值钱？显然应该从最贵的事物吃起，吃完了最贵的再吃第2贵的，这就是贪心法。
  • 如果每个物体不可分割，称为0/1背包问题。仍以吃自助餐为例，这次食物都是一份份的，每一份必须吃完。如果最贵的食物一份就超过了你的饭量，那只好放弃。这种问题无法用贪心法解决。

最长公共子序列（Longest Common Subsequence，LCS）

求两个序列的最长公共子序列

用L[i][j]表示子序列Xi 和 Yj 的最长公共子序列的长度。

  • 当Xi = Yj 时，找出Xi-1 和 Yj-1 的最长公共子序列，然后在其尾部加上Xi 即可得到X和Y的最长公共子序列。
  • 当Xi ≠ Yj 时，求解两个子问题：①求Xi-1 和 Yj 的最长公共子序列； ②求Xi 和 Yj-1 的最长公共子序列。然后取其中的最大值。
  • 可以简单理解为，当Xi = Yj 时，L[i][j]值为对角线加一，否则为上边的和左边的取较大值。

变式：最长特殊序列：

给你两个字符串 a 和 b，请返回 这两个字符串中 *最长的特殊序列* 的长度。如果不存在，则返回 -1 。「最长特殊序列」** 定义如下：该序列为 某字符串独有的最长子序列（即不能是其他字符串的子序列） 。

这题其实有点脑筋急转弯的意思。

细细分析可以知道，只有当两个字符串相同的时候会返回-1。

既然想要序列最长，那肯定想要最长的字符串长度。而如果两个字符串长度不一，那么较长的字符串本串就一定不会是较短字符串本串的子序列。所以也就可以直接返回长串的长度。

最大连续子序列和

给定一个数字序列A1、A2 、……、An，求i,j（1≤i≤j≤n），使得Ai + … + Aj 最大，输出最大和。

样例： -2 11 -4 13 -5 -2

显然11 + （-4） + 13 = 20为最大和的选取情况，因此最大和20

这里可以令状态dp[i]表示以A[i]作为末尾的连续序列的最大和。设置了这样一个dp数组，要求的最大和其实就是这个数组中的最大值，下面就是想办法求解dp数组。

如上考虑会有两种情况：

  • 这个最大和的连续序列只有一个元素，即以A[i]开始，以A[i]结尾。
  • 这个最大和的连续序列有多个元素，即从前面某处A[p]开始（p<i），一直到A[i]结尾。

对第一种情况，最大和就是A[i]本身，第二种情况最大和是dp[i-1] + A[i]。于是便可得到状态转移方程：

变式：最大子序列交替和

一个下标从 0 开始的数组的 交替和 定义为 偶数 下标处元素之 和 减去 奇数 下标处元素之 和 。

  • 比方说，数组 [4,2,5,3] 的交替和为 (4 + 5) - (2 + 3) = 4 。

给你一个数组 nums，请你返回nums中任意子序列的 最大交替和 （子序列的下标 重新 从 0 开始编号）。

一个数组的 子序列 是从原数组中删除一些元素后（也可能一个也不删除）剩余元素不改变顺序组成的数组。比方说，[2,7,4] 是 [4,2,3,7,2,1,4] 的一个子序列（加粗元素），但是 [2,4,2] 不是。

这题较之前者有两点不同：一是要求交替和；二是序列不再连续。

可以确定状态dp[i]为以i结尾的子序列最大和，但是会有这个子序列是奇还是偶的分别。

那么难点便来到了寻找状态转移方程。

依旧要分两种情况讨论，毕竟这里的交替和涉及了奇偶的情况：

  • 以i结尾时序列长度为偶数的最大交替和dp偶[i]。此时又分是否使用了这个数两种情况：
    • 使用了nums[i]，则这个数在偶数长度下为奇数下标，应当被不使用此数时的状态（即奇数长度）减去，所以值是dp奇[i-1] - nums[i]
    • 未使用nums[i]，则值应该是i-1时的偶数长度取值，也即dp偶[i-1]
    • 是故最终取值应该是上述两种情况的较大值
  • 以i结尾时序列长度为奇数的最大交替和dp奇[i]。此时仍分是否使用了这个数两种情况：
    • 使用了nums[i]，则这个数在奇数长度下为偶数下标，应当被不使用此数时的状态（即偶数长度）加上，所以值是dp偶[i-1] + nums[i]。另需注意的是，长度为奇数也可以是当前的数自成一列，所以还需要考虑取值nums[i]自身。于是这种情况便是这两个取值的较大值。
    • 未使用nums[i]，则值应该是i-1时的奇数长度取值，也即dp奇[i-1]
    • 是故最终取值应该是上述两种情况的较大值
  • 因为这个状态需要循环到数组最后一个元素，于是最后的取值便是dp奇[n-1]和dp偶数[n-1]的较大值。但稍一思索便可知，拥有最大交替和的子序列的最后一个元素一定不可能位于奇数下标也即偶数长度的序列。（因为奇数下标对应着减去该元素的值，我们完全可以不选择该元素，直接让子序列在倒数第二个偶数下标的位置结束）。所以可以直接返回dp奇[n-1]

边界：这题的起始条件便是nums[0]，而这个数只可能是奇数长度，是故可以初始化dp奇[0] = nums[0]和dp偶[0] = Integer.MIN_VALUE。

最长递增子序列(Longest Increasing Sequence,LIS)

给你一个整数数组 nums，找到其中最长严格递增子序列的长度。

子序列 是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。例如，[3,6,2,7]是数组 [0,3,1,6,2,2,7]的子序列。

令dp[i]表示以nums[i]结尾的最长递增子序列长度。于是便有两种可能：

  • 如果存在nums[i]之前的元素nums[j]（j<i），使得nums[j]≤nums[i]且dp[j] + 1 ≥ dp[i]（即把nums[i]跟在以nums[j]结尾的LIS后面时能比当前以nums[i]结尾的LIS长度更长），那么就把nums[i]跟在以nums[j]结尾的LIS后面，形成一条更长的LIS。于是又dp[i] = dp[j] + 1。
  • 如果nums[i]之前的元素都比nums[i]大，那么nums[i]只好自己形成一条LIS，长度为1.
  • 所以状态转移方程便是dp[i] = max{1,dp[j] + 1}

边界：边界已被上述状态转移方程隐含，便即是每个数都是一个LIS，即dp[i] = 1。

聪明的老鼠

沿街有一排房屋，⼀个聪明的小老鼠，计划偷窃沿街的房屋里面的粮食，每间房内都藏有⼀定的粮食。相邻的房屋装有相互连通的防盗系统， 如果两间相邻的房屋在同⼀晚上被小老鼠闯⼊，系统会⾃动报警。 给定⼀个代表每个房屋存放粮食数量的⾮负整数数组，计算你在不触动警报装置的情况下，能够偷窃到的最多粮食。（力扣相同题）

输入一个数组，输出一个数

分析一下可以知道，这里是否偷窃一个房屋是一个选择，存在偷或不偷，那么状态就应该是走到当前房屋时已经偷到的粮食数量。用dp[i]来表示到达第i个房屋时已偷到的粮食，则对于当前的第i个房屋，有两种选择：

  • 偷，那么就不能偷第i-1个房屋或者说不能在第i-1个房屋做选择，只能在第i-2个房屋前做选择。于是有dp[i] = dp[i-2] + nums[i]；
  • 不偷，那么就取决于在第i-1个房屋前所做的选择。于是有dp[i] = dp[i-1]。
  • 那么最终的状态转移方程就是dp[i] = max{dp[i-2] + nums[i], dp[i] = dp[i-1]}

变式：当房屋不再是一排，而是一圈的时候。（力扣同题）
